개발을 하다 보면 언어나 프레임워크가 당연히 제공할 것이라고 굳게 믿고 넘어가는 기본 상식들이 있습니다. 저에게는 '기본 정렬(Sort) 알고리즘은 당연히 안정 정렬(Stable Sort)일 것이다'라는 믿음이 그랬습니다.

최근 Dart(Flutter)로 2차 정렬(다중 조건 정렬) 로직을 구현하다가 이 믿음이 산산조각 났습니다. 분명 로직에는 문제가 없는데 정렬 결과가 자꾸만 뒤죽박죽이 되는 기현상을 겪었고, 무려 1시간 넘게 원인을 찾다 충격적인 사실을 알게 되었습니다.

제가 이렇게 오랜 시간 헤맸던 가장 큰 이유는 **'언어 자체를 전혀 의심하지 않았기 때문'**입니다. 거의 대부분의 현대 언어들이 기본적으로 안정 정렬을 채택하고 있으니, 당연히 Dart도 다르지 않을 것이라고 철석같이 믿고 제 로직만 계속 들여다본 것이 화근이었습니다.

## 문제 상황: 2차 정렬이 왜 안 되지?

보통 다중 조건을 기준으로 리스트를 정렬할 때 직관적인 방법 중 하나는 정렬을 두 번 연달아 수행하는 것입니다. 예를 들어 2순위 조건으로 먼저 정렬한 뒤, 1순위 조건으로 다시 정렬을 덮어씌우는 식이죠. 

대부분의 언어들은 안정 정렬(동일한 값일 경우 원래의 순서가 보장되는 정렬)을 지원하기 때문에 이 방식이 잘 작동합니다. 하지만 Dart에서 `List.sort()`를 연달아 두 번 호출했더니, 이전에 정렬해 두었던 순서가 완전히 망가지는 현상이 발생했습니다.

## 원인: Dart의 기본 정렬은 '불안정 정렬'이다

공식 문서를 찾아본 결과, **Dart의 기본 `List.sort()` 메서드는 안정 정렬을 보장하지 않는다**는 것을 알게 되었습니다.

> "The sort function is not guaranteed to be stable, so distinct objects that compare as equal might change their relative order."

Dart는 내부적으로 성능을 위해 불안정 정렬 방식을 채택하고 있습니다. 따라서 같은 우선순위를 가진 원소들이라도 정렬 후 원래의 상대적인 위치가 그대로 유지될 것이라고 기대해서는 안 됩니다.

## 해결 방법: collection 패키지

해결 방법은 아주 뻔하고 간단했습니다. 안정 정렬이 꼭 필요한 상황이라면 Dart 공식 팀에서 제공하는 `collection` 패키지의 `mergeSort`를 사용하면 됩니다.

```dart
import 'package:collection/collection.dart';

// 기존의 불안정한 myList.sort() 대신 아래와 같이 사용
mergeSort(myList, compare: (a, b) => a.compareTo(b));
```

`mergeSort`는 병합 정렬 알고리즘을 사용하므로 안정 정렬을 완벽하게 보장합니다. 이를 적용하자마자 1시간의 고생이 무색하게 다중 정렬 문제가 깔끔하게 해결되었습니다.

## 마무리

사실 이 삽질을 통해 얻은 거창한 아키텍처적 교훈 같은 것은 없습니다. 그저 **"아니, 요즘 시대에 아직도 기본 정렬로 불안정 정렬을 쓰는 언어가 있네?"** 하는 묘한 배신감, 그리고 **"대체 Dart 팀은 왜 이걸 안정 정렬로 안 바꾸는 걸까?"** 하는 의문만이 남았을 뿐입니다. 😅

혹시라도 저처럼 Dart에서 `.sort()`를 여러 번 호출하며 다중 정렬을 시도하다가 순서가 꼬이는 분이 계신다면, 로직을 의심하기 전에 언어의 기본 정렬 방식을 먼저 확인하시고 빠르게 `collection` 패키지를 꺼내 드시길 바랍니다.
