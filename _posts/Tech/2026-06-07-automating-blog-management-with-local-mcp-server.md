오늘은 제 로컬 Mac 환경에서 동작하는 지킬(Jekyll) 블로그를 AI가 제어할 수 있도록 **순수 로컬 MCP 서버**를 구축한 경험을 공유하고자 합니다.

## 왜 REST API나 CLI가 아닌 MCP인가?

일반적으로 시스템 자동화를 생각하면 가장 먼저 `REST API` 서버를 띄우거나 `CLI` 도구를 만드는 것을 떠올립니다. 

물론 REST API나 CLI 모두 훌륭한 기술이며, 외부 연동이 필요하거나 사용자가 직접 터미널을 다뤄야 할 때는 필수적입니다. 즉, 이들은 MCP와 대척점에 있는 기술이 아니라 독립적인 기술이며 상황에 따라 필요하면 사용할 뿐입니다.

하지만 제 블로그 관리 환경을 분석해보면 이야기가 조금 다릅니다. 오직 "내 로컬 PC에서, 나만의 AI 에이전트가 내 블로그를 직접 관리하게" 만드는 것이 목적이기 때문입니다. 

저는 굳이 외부 통신을 위한 REST API를 억지로 열거나, 사람이 직접 치기 위한 CLI를 별도로 개발할 필요가 없었습니다. 따라서 외부 개입 없이 오직 깃허브 CLI(`gh` 또는 `git`)를 내부적으로 통제하며 AI와 직접 소통하는 **단일 MCP 서버**를 띄우는 것이 현재 상황에 가장 적절하고 우아한 해결책이라고 판단했습니다.

## 아키텍처 및 구현

Node.js 환경에서 공식 `@modelcontextprotocol/sdk`를 사용하여 간단하게 서버를 구축했습니다.

### 1. `list_blog_categories` 도구
AI가 글을 쓸 때 기존 카테고리 구조를 파악할 수 있도록, `_posts/` 디렉토리를 읽어와 `news, tech, philosophy, etc` 등의 기존 폴더 목록을 반환합니다. 이로 인해 AI가 임의로 존재하지 않는 카테고리를 생성하는 현상을 방지합니다.

### 2. `create_blog_post` 도구
AI가 제목과 내용만 제공하면, 백엔드에서 필요한 모든 작업을 알아서 처리합니다.
- 정규식을 이용해 제목에서 특수문자를 제거하고 깔끔한 URL Slug를 자동 생성합니다.
- KST(한국 표준시)를 적용하여 시차 문제 없이 정확한 발행 날짜를 계산합니다.
- 지킬(Jekyll)의 Frontmatter(`layout`, `title`, `date`, `category`) 양식에 맞춰 마크다운 텍스트를 포매팅합니다.
- `child_process.execFileSync`를 통해 `git add`, `commit`, `push`를 순차적으로 실행하여 깃허브에 완벽하게 자동 배포합니다.
