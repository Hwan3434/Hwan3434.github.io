최근 AI 에이전트 시대가 열리면서 기존의 REST API 연동 방식을 넘어서는 새로운 패러다임이 등장했습니다. 바로 **MCP(Model Context Protocol)**입니다.

오늘은 제 로컬 Mac 환경에서 동작하는 지킬(Jekyll) 블로그를 AI가 완벽하게 제어할 수 있도록 **순수 로컬 MCP 서버**를 구축한 경험을 공유하고자 합니다.

## 왜 REST API나 CLI가 아닌 MCP인가?

일반적으로 자동화를 생각하면 `Python`으로 CLI 도구를 만들거나 `REST API` 서버를 띄우는 것을 떠올립니다. 하지만 오직 "내 로컬 PC에서 AI가 내 블로그를 관리하게" 만드는 것이 목적이라면 이야기가 달라집니다.

1. **REST API 무용론:** 외부 클라이언트나 웹 브라우저가 접속할 일이 없으므로 HTTP 라우팅이나 복잡한 인증(OAuth) 로직을 짤 필요가 없습니다.
2. **CLI 무용론:** 내가 직접 터미널에 명령어를 치는 것이 아니라, Cursor나 Claude Desktop 같은 'AI 샌드박스'가 블로그를 조작해야 하므로 AI 전용 프로토콜이 필요했습니다.

결국 핵심 로직을 감싼 **단일 MCP 서버**를 글로벌하게 띄우는 것이 가장 우아하고 완벽한 해결책이었습니다.

## 아키텍처 및 구현

기존 파이썬(3.9) 환경의 의존성 충돌을 피해, 최신 Node.js 환경에서 공식 `@modelcontextprotocol/sdk`를 사용하여 구축했습니다.

### 1. `list_blog_categories` 도구
AI가 글을 쓸 때 기존 카테고리 구조를 파악할 수 있도록, `_posts/` 디렉토리를 읽어와 `news, tech, philosophy, etc` 등의 기존 폴더 목록을 반환합니다. 이로 인해 AI가 마음대로 엉뚱한 카테고리를 생성하는 '환각(Hallucination)' 현상을 막습니다.

### 2. `create_blog_post` 도구
AI가 제목과 내용만 던져주면, 백엔드에서 모든 지저분한 작업을 처리합니다.
- 정규식을 이용해 제목에서 특수문자를 제거하고 깔끔한 URL Slug를 자동 생성합니다.
- KST(한국 표준시)를 적용하여 시차 문제 없이 정확한 발행 날짜를 계산합니다.
- 지킬(Jekyll)의 Frontmatter(`layout`, `title`, `date`, `category`) 양식에 맞춰 마크다운 텍스트를 포매팅합니다.
- `child_process`를 통해 `git add`, `commit`, `push`를 한 번에 실행하여 깃허브에 자동 배포합니다.

## 결과

이제 어떤 프로젝트 폴더에서 어떤 코딩을 하고 있든 간에, "이거 블로그 기술 탭에 올려줘!" 한 마디면 백그라운드에 떠 있는 로컬 MCP 서버가 모든 동작을 원스톱으로 처리합니다.

이러한 **Core Logic + MCP** 아키텍처는 향후 AI 중심(AI-Native) 백엔드 생태계에서 가장 기본적이고 강력한 설계 패턴이 될 것입니다! 🚀
